## ゲーム制作講座

# コンパイラとは

C++に置けるコンパイルとは、ソースコードが書かれたファイル(.cpp)を機械が読むための機械語に翻訳することをコンパイルするという。  
また、このプログラムをコンパイラという。  

ソースファイルをコンパイルして生成されるファイルをオブジェクトファイル(.o, .obj)という。  
このオブジェクトファイルは外部リンケージを持つ識別子の名前情報を保持したままである。  
(詳細はリンケージの回で)  

通常コンパイルはcppファイルごとに行われる。  
一度でコンパイルされる単位のことを翻訳単位という。  

# リンカ

C++に置けるリンカとは、コンパイルによって生成されたオブジェクトファイルやライブラリを結合し、実行可能なプログラムを作成するプログラムのことである。  
リンケージエディタとも呼ばれる。  

ユーザ定義された関数名などはこの時に破棄される。  

# プロジェクトとビルド

Visual Studio にはプロジェクトやソリューションといった概念がある。  

プロジェクトはソースコードが書かれたファイルなどをまとめて管理する。  
プロジェクトをビルドするとコンパイラ、リンカがそれぞれ動き、実行ファイルを生成する。  

Ctrl + F5 を押すとビルドして生成されたファイルを実行している。  
単にコンパイルと行ったとき、このビルドを指していることもあるので違いに注意が必要だ。

# ソリューション

Visual Studio にはソリューションという概念がある。  
ソリューションは複数のプロジェクトをまとめて管理するためのものだ。  

ソリューションにプロジェクトを追加して全く別のプログラムを作成、実行することができる。
